COmPOSER

COmPOSER is an RF/mm-wave design automation framework for performance-oriented synthesis. In the paper, it is presented as an open-source end-to-end flow that translates target specifications into optimized circuits with layouts, unifying schematic synthesis, primitive layout generation, placement, routing, and layout-aware passive modeling.

The repository is organized so that you can use the full pipeline or only the pieces you need. You can run the end-to-end shell driver, or you can call the sizers, parser, placement engine, router, primitive generators, and utilities independently in your own project.

Paper-aligned framing

These pages now follow the attached paper's framing of COmPOSER as a layout-aware RF/mm-wave synthesis framework, not just a collection of scripts.

The site reflects the actual public repo structure while also matching the paper's language around hybrid sizing, primitive generation, layout synthesis, and verification.

Main Ideas

Paper Stage 1

Hybrid analytical sizing uses circuit equations plus ML-informed passive estimation to produce layout-aware active and passive targets.

Paper Stage 2

Primitive block layout generation turns sized devices, passives, CPWs, and matching structures into reusable DRC-clean layout blocks.

Paper Stage 3

Layout synthesis combines constraint-driven MILP placement, A*-based routing, and PDN generation with user RF priorities.

Paper Stage 4

Post-layout verification closes the loop between synthesis intent and extracted behavior. The public repo exposes the synthesis flow that feeds that validation story.

Abstract-Aligned Summary

Based on the attached paper PDF, COmPOSER is positioned as a framework that:

  • translates target specifications into optimized RF/mm-wave circuits with layouts
  • unifies schematic synthesis, active and passive layout generation, and placement and routing
  • combines physics-based equations with machine-learning-driven electromagnetic models
  • targets LNA and PA design classes
  • emphasizes productivity gains relative to manual design

Public Repo and Paper Scope

The paper reports results validated in a commercial 65 nm environment and discusses post-layout verification with commercial EM and SPICE tools.

This public repository contains the open synthesis flow, example assets, mock datasets, mock PDK files, and the routing and layout-generation infrastructure. The website now makes that distinction explicit so the docs match the paper without overstating what is bundled locally.

Important Notes

The CSV files in DATASETS/ are reference-format datasets, not production datasets. Users should generate their own datasets with the same column names and the same units expected by the scripts.

The MODELS/ directory is intended to hold trained .pkl artifacts, but those model files are not committed. They are created locally during training or setup.

The mock PDK and fixed primitive GDS files are flow-compatible placeholders. They are useful for validating the public pipeline, but users should substitute technology-correct assets for real deployment.
